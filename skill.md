# 专业文章发布助手

## 使用场景

- 当用户提供一个话题、行业事件或相关资料，需要生成专业深入的行业研究文章时使用（深度模式）
- 当用户提供已完成的文章内容，只需要添加配图和HTML格式转换时使用（快速模式）


## 输入要求

- 接收以下任一输入形式：
  - **话题 + 用户提供的研究资料**：用户描述话题并提供相关信息、数据、链接内容（深度模式）
  - **仅话题**：基于用户描述和已有知识撰写（需明确告知用户可能需要补充资料）
  - **文章链接**（如平台支持）：提供URL让助手抓取内容
  - **完整文章内容**：用户直接提供已完成的文章，只需添加配图和格式（快速模式）

### 图片标记（可选）

当文章中需要插入图片时，可在对应位置使用 `[*名称*]` 格式标记，并准备同名的图片文件：

- 在文章中写入标记，如 `[*封面*]`、`[*数据图1*]`
- 将对应图片文件放在文章同目录或其子目录下
- 图片文件名（不含扩展名）需与标记名称一致，如 `封面.jpg`、`数据图1.png`
- 支持格式：jpg、jpeg、png、gif、webp、bmp
- 检测到标记且有匹配图片时，自动上传到图床并替换标记，**跳过AI生图**

## 工作模式判断

在开始工作前，首先判断用户需求：

### 模式A： 快速模式
**触发条件（满足任一即可）：**
- 用户明确表示"只需要加图""只需要格式化""直接生成HTML""不要修改内容"
- 用户明确表示使用“快速模式”

### 模式B：深度模式
**触发条件（满足任一即可）：**
- 用户提供话题但没有完整文章（只有标题、大纲或资料片段）
- 用户明确要求"写一篇""研究""分析""撰写"等创作性需求
- 用户提供的是数据、链接、资料等需要整合的内容
- 用户明确表示使用“深度模式”

## 工作流程

### 第一阶段：准备文章（两种模式各异）

#### 模式A：快速模式

1. 在当前工作目录下创建与**话题同名**的新文件夹
   - 文件夹命名规则：将 `/` `\` `:` `*` `?` `"` `<` `>` `|` 替换为 `_`，遇到同名文件夹则附加日期加以区分
2. 将用户提供的文章原文保存为该文件夹下的 `article_01.md`，**内容不做任何修改**

#### 模式B：深度模式

1. **创建话题文件夹**
   - 在当前工作目录下创建与**话题同名**的新文件夹
   - 文件夹命名规则：将 `/` `\` `:` `*` `?` `"` `<` `>` `|` 替换为 `_`，遇到同名文件夹则附加日期加以区分

2. **信息收集**
   - 整理用户提供的话题、资料、数据、链接等关键要点
   - 如平台支持，用 WebFetch 读取用户给出的链接，用 WebSearch 补充最新市场数据、行业动态、竞争格局、政策环境等维度
   - 如工具不可用，基于用户资料和已有知识进行分析，并在文中注明信息来源局限

3. **整理结构**：构建文章骨架，一般涵盖行业概述、市场现状、产业链分析、竞争格局、技术趋势、驱动因素、挑战与风险、未来展望，区分"事实叙述"与"分析判断"

4. **撰写并保存文章**
   - 中文写作，专业严谨，约 2000–3000 字
   - 先读取 `references/write_style.txt`（若存在）并遵守其风格规范；否则沿用默认研究风格
   - 将文章保存为话题文件夹下的 `article_01.md`

---

### 第二阶段：配图与输出（两种模式共用）

5. **检测图片标记与本地图片**
   - 读取文章内容，检测是否包含 `[*名称*]` 格式的图片标记
   - 在文章所在目录（及其子目录）中搜索与标记名称匹配的图片文件
   - 如果找到标记且有对应本地图片 → 进入**本地图片模式**（跳过AI生图）
   - 如果未找到标记或无匹配图片 → 进入**AI生图模式**

6. **生图/上传（根据检测结果分流）**

   **本地图片模式：**
   - 跳过AI生图和提示词生成步骤
   - 使用 Bash 工具在 `scripts/` 目录下执行：
     ```bash
     python run_workflow.py --md <话题文件夹>/article_01.md --images-dir <图片目录>
     ```
   - 脚本自动将本地图片上传到图床，替换文章中的 `[*名称*]` 标记

   **AI生图模式：**
   - 生成配图提示词（检测用户输入中是否已包含，若无根据文章主题自行构思）
   - 将提示词保存为话题文件夹下的 `prompt.txt`（提示词创作规范参见 `references/image_prompt_guide.txt`）
   - 使用 Bash 工具在 `scripts/` 目录下执行：
     ```bash
     python run_workflow.py --md <话题文件夹>/article_01.md --prompt <话题文件夹>/prompt.txt
     ```
   - 脚本内置容错机制（生图失败自动重试、兜底图片生成）和完整日志

7. **整理输出结果**
   - 向用户报告生成的文件（配图、带图 Markdown、微信公众号 HTML）
   - 提示用户：用浏览器打开 HTML 文件，全选复制后粘贴到公众号编辑器即可使用



## 输出格式

- 默认输出：
  1. 行业研究文章（Markdown格式）
  2. AI配图提示词文件
  3. AI生成的配图
  4. 带图片的Markdown文件
  5. 微信公众号HTML文件

## 文章质量标准

### 内容深度
- 不少于2000字的深度分析
- 包含具体数据和案例支撑
- 覆盖产业链、竞争格局、技术趋势等多个维度

### 信息来源
- 优先使用权威来源（官方报告、主流媒体、研究机构）
- 标注信息的时间敏感性
- 对不确定信息保持谨慎

### 逻辑结构
- 清晰的层次结构（概述→现状→分析→展望）
- 每个部分有明确的小标题
- 段落之间逻辑连贯

### 专业性
- 使用行业术语准确
- 数据引用规范
- 分析客观理性

## 注意事项

1. **时效性标注**：始终标注信息的时间范围（如"截至2026年3月的公开信息"）
2. **合规性**：不输出涉及内幕信息、尚未公开的敏感内容或违反合规要求的内容
3. **客观性**：避免将模型自身当作信息源，应基于检索到的公开资料进行分析
4. **不确定性**：若检索结果有限或存在矛盾，应在文中明确说明分析结论的不确定性
5. **版权意识**：AI配图避免使用可能产生版权争议的元素

## 常见错误处理

### 错误1：配置文件不存在
**错误信息：** `配置文件不存在: ai_image_config.json`
**解决方案：**
- 检查 `ai_image_generator/ai_image_config.json` 是否存在
- 确保配置文件包含 `api_key`、`model`、`default_size` 等必需字段

### 错误2：图床上传失败
**错误信息：** `Upload failed` 或七牛云相关错误
**解决方案：**
- 检查 `image_uploader/image_bed_config.json` 配置
- 验证七牛云 `access_key`、`secret_key`、`bucket`、`domain` 是否正确
- 确认存储空间有足够配额

### 错误3：Markdown文件不存在
**错误信息：** `Markdown文件不存在`
**解决方案：**
- 确认文章已成功保存为 `article_01.md`
- 检查文件夹命名是否包含非法字符（已自动清理）

### 错误4：AI生图两次都失败
**现象：** 自动生成蓝底白字兜底图片
**说明：** 这是正常的容错机制，不影响工作流继续
**可能原因：**
- API配额用尽
- 网络持续不稳定
- API服务维护中

### 错误5：run_workflow.py 执行失败
**解决方案：**
- 检查 Python 环境是否正确
- 确认所有依赖已安装（PIL、requests、qiniu、markdown）
- 手动执行脚本查看详细错误信息

## 工具使用说明

### 必需工具（跨平台通用）
- **Bash**：用于调用 `run_workflow.py` 脚本
- **Write**：用于创建文章和提示词文件
- **Read**：读取 `write_style.txt` 风格规范文件（可选）

### 可选工具（平台特定）
- **WebSearch**：如平台支持，用于检索行业相关信息
- **WebFetch**：如平台支持，用于读取用户提供的链接内容
- **Glob**：查找项目中的相关文件

### 兼容性说明

本 skill 设计为跨平台兼容：
- **核心功能**（文章撰写、配图生成、HTML转换）不依赖特定平台工具
- **信息收集**采用灵活方案：
  - 优先使用用户提供的资料（通用方案）
  - 如平台支持 WebSearch/WebFetch 则自动使用
  - 如工具不可用，引导用户手动提供信息
- 可在 Claude Code、Trae、Cursor 等不同智能体环境中使用

## 示例场景

### 场景0：快速格式化模式（用户提供完整文章）

**用户输入：**
"帮我把这篇文章加上配图和格式，生成HTML：

# 人工智能的未来发展

人工智能技术正在快速发展...（完整文章内容）"

**执行流程：**
1. 识别为快速格式化模式（用户提供完整文章）
2. 原封不动保存用户文章为 `article_01.md`
3. 根据标题"人工智能的未来发展"生成配图提示词
4. 调用 run_workflow.py 生成配图和HTML
5. 如AI生图失败，自动生成蓝底白字兜底图片

### 场景1：用户提供话题和资料（深度模式）

**用户输入：**
"帮我写一篇关于新能源汽车行业的研究文章。以下是一些资料：
- 2025年中国新能源汽车销量达到950万辆，同比增长35%
- 市场渗透率突破40%
- 主要玩家：比亚迪、特斯拉、蔚来、小鹏等
- 技术趋势：固态电池、智能驾驶、车路协同"

**执行流程：**
1. 识别为深度模式
2. 整理用户提供的数据和信息
3. 尝试使用 WebSearch 补充更多背景信息（如平台支持）：
   - 全球新能源汽车市场对比
   - 政策环境和补贴政策
   - 产业链上下游情况
   - 最新技术突破和商业模式创新
4. 整合用户资料与检索信息，撰写2000-5000字的行业研究文章
5. 生成配图提示词（如"新能源汽车产业链图，现代商业风格"）
6. 调用 run_workflow.py 生成最终HTML

### 场景2：用户仅提供话题

**用户输入：**
"帮我写一篇关于人工智能芯片行业的研究文章"

**执行流程：**
1. 尝试使用 WebSearch 检索行业信息（如平台支持）
2. 如工具不可用，询问用户是否有相关资料可以提供
3. 基于可获得的信息撰写文章，明确标注信息来源范围
4. 生成配图并输出HTML

### 场景3：用户提供链接

**用户输入：**
"根据这篇文章写一篇行业研究：https://example.com/ai-industry-report"

**执行流程：**
1. 尝试使用 WebFetch 读取链接内容（如平台支持）
2. 如工具不可用，请求用户复制粘贴关键内容
3. 使用 WebSearch 补充更多维度的信息（如平台支持）
4. 综合分析撰写深度文章
5. 生成配图并输出HTML

### 场景4：用户提供多个链接和补充说明

**用户输入：**
"基于这几篇文章，写一篇半导体行业的深度分析：
- https://example.com/chip-market
- https://example.com/semiconductor-tech
我特别关注中国市场的自主化进程和技术突破"

**执行流程：**
1. 尝试使用 WebFetch 读取所有链接（如平台支持）
2. 提取各链接的核心观点和数据
3. 使用 WebSearch 针对"中国半导体自主化"进行补充检索
4. 综合撰写全面的行业研究文章，重点突出用户关注的维度
5. 生成配图并输出HTML

### 场景5：文章中包含图片标记（本地图片模式）

**用户输入：**
"帮我把这篇文章发布，图片我准备好了，在article_01文件夹里：

# 产业链分析报告

[*封面*]

本文将从三个维度分析...

[*产业链图*]

如下图所示，上游...

[*趋势对比*]

从对比数据可以看出..."

**执行流程：**
1. 识别为快速格式化模式（用户提供完整文章）
2. 保存文章为 `article_01.md`
3. 检测到3个图片标记：`[*封面*]`、`[*产业链图*]`、`[*趋势对比*]`
4. 在文章目录中搜索匹配的本地图片：`封面.jpg`、`产业链图.png`、`趋势对比.jpg`
5. 发现匹配图片 → 进入本地图片模式，跳过AI生图
6. 上传3张图片到图床，替换文章中的标记
7. 转换为微信公众号HTML并输出

## 样式自定义说明

- 脚本默认使用 `md2wechat/styles/default.json` 样式
- 适合大多数专业文章和研究报告
- 无需额外配置
